## Active redesign: Ancient Greek Atrium (Three.js)

The entire portfolio is being rebuilt as an immersive 3D ancient Greek atrium.
Visitor enters a marble hall with Corinthian columns, coffered ceiling, and project panels on the walls.

### The .glb scene (already built)
`public/models/atrium.glb` was generated by `build_atrium.py` (Blender 5.1 headless).
It contains:
- Room shell (floor, 4 walls, ceiling) — 24m × 10m × 7m
- 8 Corinthian columns with fluted shafts (4 per side)
- Coffered ceiling (3×8 recessed panel grid)
- 6 project panel frames + canvas planes (tagged with `panel_id` and `is_panel_canvas`)
- 6 statue plinths between columns
- 2 entablature beams (one per side wall)
- Camera + lights (supplemented by Three.js at runtime)

### Panel canvas objects
The GLB contains flat planes named `Panel_{id}_canvas` with custom properties:
```
userData.is_panel_canvas = true
userData.panel_id = 'ema' | 'artsy' | 'augusts' | 'academia' | 'yummy' | 'thessaloniki'
```
At runtime, `AtriumScene.vue` replaces their material with a `THREE.CanvasTexture`
built from the `buildPanelTexture()` function (project title, tags, screenshot, buttons).

## Three.js conventions used in this project

- **No `@tresjs`** — raw Three.js only, instantiated in `onMounted`, cleaned up in `onBeforeUnmount`
- `GLTFLoader` from `three/examples/jsm/loaders/GLTFLoader`
- `OrbitControls` from `three/examples/jsm/controls/OrbitControls`
- `RGBELoader` from `three/examples/jsm/loaders/RGBELoader` (HDRI)
- All panel canvases are `THREE.CanvasTexture` built at runtime via the `buildPanelTexture()` function
- Hover detection via `THREE.Raycaster` in the animation loop
- Camera movement uses a manual lerp loop (no GSAP dependency)

## Agent instructions

When working on this repo:

1. **Never use `bpy.ops` in Blender scripts** — Blender 5.x background mode requires pure `bpy.data` / `bmesh` API
2. **Never open the Blender GUI** — always use `--background --python`
3. **Always clean up Three.js** in `onBeforeUnmount` — cancel `requestAnimationFrame`, call `renderer.dispose()`
4. **Panel canvases are identified by `userData.is_panel_canvas === true`** — traverse the GLB scene and check this flag
5. **Nuxt 3 + Vue 3 only** — no Options API, no Vue 2 patterns
6. **Tailwind for any 2D UI** — the modal, HUD, loading screen all use Tailwind classes
7. **TypeScript** — all new `.ts` / `.vue` files should be typed
8. **Do not break existing pages** — the 3D scene is only on `pages/index.vue`; other pages stay untouched
9. **Netlify static** — no server-side routes, no API routes, everything is client-side
